# ppx_uopt

`ppx_uopt` derives allocation-free `Option`-style helpers for unboxed scalar types and
unboxed records in OCaml.

It generates an `Option` module with:
- `none`
- `some`
- `is_none`
- `is_some`
- `value`
- `value_exn`
- `unchecked_value`
- `Optional_syntax`

The generated code is intended for sentinel-based unboxed representations, so the PPX
stays explicit about where `none` comes from and how `is_none` is checked.

## Supported Types

Supported scalar types:
- `float#`
- `float32#`
- `int32#`
- `int64#`
- `bits64`
- `nativeint#`
- `int8#`
- `int16#`
- `int#`
- `char#`

Supported record types:
- unboxed records whose fields are all either:
  primitive supported unboxed scalar fields
- or module-qualified `M.t` contract fields

For nested custom fields, only `M.t` is supported. The PPX does not infer option
semantics from representation alone.

## Contract Fields

A record field of type `M.t` is supported only when the generated code can rely on this
explicit contract:

```ocaml
M.Option.none : M.t
M.Option.is_none : M.t -> bool
```

If a field is written as `M.t`, `ppx_uopt` will generate code that refers explicitly to:
- `M.Option.none`
- `M.Option.is_none`

It does not assume that every module-qualified type has an `Option` module.

## Examples

Scalar with default sentinel:

```ocaml
type token = float# [@@deriving unboxed_option]
```

Scalar with explicit sentinel override:

```ocaml
type token = int8# [@@deriving unboxed_option { none = #0s }]
```

Record with explicit sentinel override:

```ocaml
type packed_pair =
  #{ x : int8#
   ; y : int32#
   }
[@@deriving unboxed_option { none = #{ x = #0s; y = #0l } }]
```

Record with nested contract field:

```ocaml
module Foo = struct
  type t = float#

  module Option = struct
    let none = Float_u.nan ()
    let is_none t = Float_u.is_nan t
  end
end

type record = #{ x : Foo.t } [@@deriving unboxed_option]
```

## Notes

- Some scalar types require an explicit `none = ...` override because they do not have a
  safe default sentinel.
- Unboxed records may also supply an explicit record-level `none = #{ ... }` override.
  This is required when one or more scalar fields do not have a default sentinel, and it
  also allows raw unboxed literal sentinels such as:
  `[@@deriving unboxed_option { none = #{ x = #0s; y = #0l } }]`.
- For unboxed records, `is_none` is generated by OR-ing the field-level none checks.
- For contract fields, the field-level check is `M.Option.is_none t.#field`.
- For contract fields in a default record `none`, the field value is `M.Option.none`
  unless the whole record supplies an explicit `none = #{ ... }` override.
